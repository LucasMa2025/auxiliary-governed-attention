## 一、日志系统概述

AGA 提供生产级结构化日志能力，支持多种输出格式和目标。

### 1.1 核心特性

| 特性 | 说明 |
|------|------|
| **结构化日志** | JSON 格式输出，兼容 ELK Stack 和 Loki |
| **请求上下文** | 线程安全的 `LogContext`，自动注入 trace_id |
| **多输出目标** | stdout、文件、或两者同时 |
| **第三方库控制** | 自动降低 uvicorn/httpx 等库的日志级别 |
| **消息截断** | 防止超长日志影响性能 |

### 1.2 日志格式

#### JSON 格式 (生产环境推荐)

```json
{
  "timestamp": "2026-02-09T10:30:45.123Z",
  "level": "INFO",
  "logger": "aga.production.operator",
  "message": "Knowledge injected successfully",
  "service": "aga-portal",
  "trace_id": "abc-123-def-456",
  "context": {
    "namespace": "production",
    "lu_id": "knowledge_001"
  },
  "source": {
    "file": "operator.py",
    "line": 245,
    "function": "inject_knowledge"
  },
  "extra": {
    "slot_idx": 5,
    "lifecycle_state": "probationary"
  }
}
```

#### 文本格式 (开发环境)

```
2026-02-09 10:30:45 | INFO     | aga.production.operator | [trace_id=abc-123 | namespace=production] Knowledge injected successfully
```

## 二、日志配置

### 2.1 配置类

```python
from aga.monitoring import LoggingConfig, setup_logging

# 创建配置
config = LoggingConfig(
    service_name="aga-portal",      # 服务名称
    level="INFO",                   # 日志级别
    format="json",                  # 格式: json, text
    output="both",                  # 输出: stdout, file, both
    log_file="/var/log/aga/portal.log",
    include_extra=True,             # 包含额外字段
    include_source=True,            # 包含源码位置
    max_message_length=10000,       # 消息最大长度
    third_party_level="WARNING",    # 第三方库日志级别
)

# 应用配置
setup_logging(config)
```

### 2.2 环境变量配置

| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| `AGA_LOG_FORMAT` | 日志格式 (json/text) | text |
| `AGA_LOG_LEVEL` | 日志级别 | INFO |
| `AGA_LOG_FILE` | 日志文件路径 | - |

### 2.3 启动脚本参数

```bash
# 开发模式 (text 格式, DEBUG 级别)
./scripts/start_api.sh --dev

# 生产模式 (json 格式, WARNING 级别)
./scripts/start_api.sh --prod

# 自定义配置
./scripts/start_api.sh --log-format json --log-level INFO --log-file /var/log/aga/api.log
```

## 三、日志上下文

### 3.1 LogContext 类

线程安全的日志上下文管理器，用于在日志中添加请求级别的上下文信息。

```python
from aga.monitoring import LogContext, log_context, generate_trace_id

# 方式 1: 直接设置
LogContext.set(
    trace_id=generate_trace_id(),
    namespace="production",
    user_id="user_001"
)

# 日志会自动包含这些上下文
logger.info("Processing request")  # 输出包含 trace_id, namespace, user_id

# 清除上下文
LogContext.clear()

# 方式 2: 上下文管理器 (推荐)
with log_context(trace_id="abc-123", namespace="production"):
    logger.info("Processing request")
    # 上下文自动清除
```

### 3.2 在 FastAPI 中使用

```python
from fastapi import FastAPI, Request
from aga.monitoring import LogContext, generate_trace_id

app = FastAPI()

@app.middleware("http")
async def add_trace_context(request: Request, call_next):
    # 从请求头获取或生成 trace_id
    trace_id = request.headers.get("X-Trace-ID") or generate_trace_id()
    
    # 设置上下文
    LogContext.set(
        trace_id=trace_id,
        path=request.url.path,
        method=request.method,
    )
    
    try:
        response = await call_next(request)
        response.headers["X-Trace-ID"] = trace_id
        return response
    finally:
        LogContext.clear()
```

## 四、辅助日志函数

### 4.1 log_request - HTTP 请求日志

```python
from aga.monitoring import log_request, get_logger

logger = get_logger("aga.api")

log_request(
    logger,
    method="POST",
    path="/api/v1/knowledge/inject",
    status_code=200,
    duration_ms=45.3,
    namespace="production",
    lu_id="knowledge_001",
)
```

输出:
```json
{
  "message": "POST /api/v1/knowledge/inject 200 45.30ms",
  "http": {
    "method": "POST",
    "path": "/api/v1/knowledge/inject",
    "status_code": 200,
    "duration_ms": 45.3
  },
  "namespace": "production",
  "lu_id": "knowledge_001"
}
```

## 五、监控指标系统

### 5.1 MetricsRegistry

统一的指标注册中心，支持 Prometheus Counter/Gauge/Histogram。

```python
from aga.monitoring import get_metrics_registry

registry = get_metrics_registry()

# 获取或创建指标
counter = registry.get_metric(
    "requests_total",
    metric_type="counter",
    documentation="Total requests",
    labelnames=["namespace", "status"]
)

# 使用指标
counter.labels(namespace="production", status="success").inc()

# 导出 Prometheus 格式
metrics_text = registry.export_text()
```

### 5.2 装饰器

```python
from aga.monitoring import track_latency, track_errors

@track_latency("forward_duration_seconds", labelnames=["namespace"])
@track_errors("forward_errors_total", labelnames=["namespace"])
def forward(self, hidden_states, namespace="default"):
    # 自动记录延迟和错误
    ...
```

### 5.3 上下文管理器

```python
from aga.monitoring import track_ann_search, track_loader

# ANN 搜索追踪
with track_ann_search(namespace="production") as ctx:
    results = index.search(query_vector, top_k=100)
    ctx.labels["candidates"] = len(results)

# 动态加载器追踪
with track_loader(namespace="production") as ctx:
    slots = loader.load_candidates(lu_ids)
    ctx.labels["hot_hits"] = slots.hot_hits
    ctx.labels["cold_loads"] = slots.cold_loads
```

### 8.2 自定义 SLO

```python
from aga.monitoring import ServiceLevelObjective, ServiceLevelIndicator

custom_slo = ServiceLevelObjective(
    name="Custom_Latency_SLO",
    description="自定义延迟目标",
    time_window="1h",
    target_compliance=0.99,
    sli_list=[
        ServiceLevelIndicator(
            name="P95Latency",
            description="P95 延迟",
            metric_expr='histogram_quantile(0.95, rate(aga_request_duration_seconds_bucket[1h]))',
            target_value=0.3,  # 300ms
            operator="<=",
            unit="seconds"
        )
    ]
)
```

## 九、告警规则

### 9.1 已定义告警

| 告警名称 | 严重级别 | 触发条件 |
|----------|----------|----------|

```yaml
  - type: log
    paths:
      - /var/log/aga/*.log
    json.keys_under_root: true
    json.add_error_key: true

output.elasticsearch:
  hosts: ["elasticsearch:9200"]
  index: "aga-logs-%{+yyyy.MM.dd}"
```

### 12.3 Prometheus 配置

```yaml
# prometheus.yml
scrape_configs:
  - job_name: 'aga'
    static_configs:
      - targets: ['aga-api:9090', 'aga-portal:9090']
    relabel_configs:
      - source_labels: [__address__]
        regex: '(.+):9090'
        target_label: instance
        replacement: '$1'

  alertmanagers:
    - static_configs:
        - targets: ['alertmanager:9093']

  - /etc/prometheus/aga_alerts.yml
```
