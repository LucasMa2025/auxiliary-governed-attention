# AGA 部署指南

本文档说明 AGA 的两种部署模式：**Demo（单机）** 和 **生产（分布式）**。

## 部署模式对比

| 特性     | Demo（单机）           | 生产（分布式）            |
| -------- | ---------------------- | ------------------------- |
| 架构     | 单进程，API + 推理一体 | Portal + Runtime 分离     |
| GPU      | 需要（推理）           | Portal 无需，Runtime 需要 |
| 持久化   | SQLite                 | PostgreSQL + Redis        |
| 消息队列 | 内存                   | Redis / Kafka             |
| 扩展性   | 单机                   | 水平扩展                  |
| 适用场景 | 开发、测试、演示       | 生产环境                  |

### 启动方式

**Windows:**

```batch
# 开发模式（自动重载）
scripts\start_api.bat --dev

# 默认模式
scripts\start_api.bat

# 指定端口
scripts\start_api.bat --port 8082
```

**Linux/Mac:**

```bash
# 开发模式
./scripts/start_api.sh --dev

# 默认模式
./scripts/start_api.sh
```

**Python 直接启动:**

```bash
# 最简启动
python -m aga.api

# 完整参数
python -m aga.api \
    --host 0.0.0.0 \
    --port 8081 \
    --hidden-dim 4096 \
    --bottleneck-dim 64 \
    --num-slots 100 \
    --persistence-type sqlite \
    --db-path ./aga_data.db
```

### 配置说明

| 参数                 | 默认值            | 说明                   |
| -------------------- | ----------------- | ---------------------- |
| `--host`             | 0.0.0.0           | 监听地址               |
| `--port`             | 8081              | 监听端口               |
| `--hidden-dim`       | 4096              | 隐藏层维度（匹配模型） |
| `--bottleneck-dim`   | 64                | 瓶颈层维度             |
| `--num-slots`        | 100               | 每命名空间槽位数       |
| `--persistence-type` | sqlite            | 持久化类型             |
| `--db-path`          | ./aga_api_data.db | 数据库路径             |
| `--reload`           | -                 | 开发模式自动重载       |

### 组件说明

#### 1. AGA Portal

Portal 是知识管理的入口，**无需 GPU**，可部署在普通服务器上。

**职责：**

-   接收外部治理系统的知识注入请求
-   管理知识元数据和生命周期状态
-   通过消息队列将变更同步到 Runtime
-   提供审计日志和统计信息

**启动方式：**

```bash
# 开发模式
scripts/start_portal.sh --dev

# 生产模式（PostgreSQL + Redis）
scripts/start_portal.sh --prod \
    --postgres postgresql://user:pass@db-host/aga \
    --redis redis-host

# 或使用配置文件
python -m aga.portal.app --config configs/portal_config.yaml
```

#### 2. AGA Runtime

Runtime 是推理执行层，**需要 GPU**，负责实际的知识注入和推理。

**职责：**

-   订阅 Portal 发布的知识变更消息
-   维护本地 AGA 模块和槽位池
-   执行 LLM 推理（带 AGA 增强）
-   向 Portal 报告状态

**启动方式：**

```python
from aga.runtime import RuntimeAgent
from aga.config import RuntimeConfig, SyncConfig

# 配置
config = RuntimeConfig(
    instance_id="runtime-001",
    portal_url="http://portal-host:8081",
    namespaces=["default", "production"],
)

sync_config = SyncConfig(
    backend="redis",
    redis_host="redis-host",
    redis_port=6379,
)

# 启动 Agent
agent = RuntimeAgent(config)
await agent.initialize(sync_config)
await agent.start()
```

### 配置文件

**Portal 配置 (`configs/portal_config.yaml`):**

```yaml
server:
    host: 0.0.0.0
    port: 8081
    workers: 4
    log_level: info

persistence:
    type: postgres
    postgres_url: postgresql://user:pass@localhost/aga
    postgres_pool_size: 10

messaging:
    backend: redis
    redis_host: localhost
    redis_port: 6379
    redis_channel: aga:sync

registry:
    type: redis
    timeout: 60

environment: production
```

**Runtime 配置 (`configs/runtime_config.yaml`):**

```yaml
instance_id: runtime-001
portal_url: http://portal-host:8081
namespaces:
    - default
    - production

sync:
    backend: redis
    redis_host: localhost
    redis_port: 6379

aga:
    hidden_dim: 4096
    bottleneck_dim: 64
    num_slots: 100
    num_heads: 32
```

## 常见问题

### Q: Demo 模式能否用于生产？

不建议。Demo 模式使用 SQLite 和内存队列，不支持水平扩展，且单点故障风险高。

### Q: 生产模式必须使用 PostgreSQL 吗？

推荐使用 PostgreSQL，但也支持 SQLite（仅适合小规模部署）。

### Q: Runtime 可以不使用 GPU 吗？

可以，但推理速度会很慢。AGA 本身的计算量不大，主要是 LLM 推理需要 GPU。

### Q: 如何从 Demo 迁移到生产模式？

1. 导出 SQLite 数据
2. 导入到 PostgreSQL
3. 部署 Portal 和 Runtime
4. 配置消息队列
5. 验证数据同步
